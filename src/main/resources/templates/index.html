Signature Demo Queries

Copy and paste these directly into GraphiQL. They best illustrate GraphQL's advantages for relational domains.

This single query traverses 5 tables with multiple relationships. 
query FullRelationalHierarchy {
  customer(id: 1) {
    id
    entityName
    type
    contacts {
      firstName
      lastName
      email
      phone
    }
    shipmentLocations {
      name
      city
      state
      zip
      gateCode
    }
    shipments {
      activity
      itemDescription
      weight
      dimensions
      contact { lastName email }
      shipmentLocation { name city }
      serviceOffering { description typeCd }
    }
  }
}
In REST this would typically require 4–6 separate API calls + client-side assembly.

Client-controlled pagination, filtering and sorting. 
query FilteredPaginatedShipments {
  shipments(
    filter: { activity: PICKUP, minWeight: 10 }
    page: 0
    size: 5
    sort: ["-weight"]
  ) {
    content {
      id
      itemDescription
      weight
      activity
      customer { entityName }
    }
    totalElements
    totalPages
  }
}

Even nested lists support pagination & sorting arguments. 
query CustomerWithLimitedShipments {
  customer(id: 1) {
    entityName
    shipments(page: 0, size: 2, sort: ["-weight"]) {
      id
      weight
      itemDescription
      serviceOffering { description }
    }
  }
}
